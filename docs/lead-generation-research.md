# B2B 营销线索问题深度调研报告

> 调研时间：2026年5月
> 调研目的：为「AI Growth Copilot」项目提供业务背景支撑，厘清 Agent 与人工的责任边界

---

## 一、背景问题概述

营销团队在产出「潜在客户线索」时，面临的核心矛盾是：

> **线索获取成本持续上升，但有效转化率持续下降**

根据行业数据：
- 市场部投放获取的线索，约 **60%-80% 是无效线索**（无需求、无预算、无决策权）
- 销售团队跟进意愿低，大量线索被浪费或错过最佳时机
- 市场与销售互相抱怨：「线索质量差」「跟进不力」成为常态

---

## 二、营销团队的核心痛点

### 2.1 线索质量问题

#### 问题1：来源碎片化，数据归因困难

企业从多个渠道获取线索（展会、官网表单、公众号、活动报名、直播等），但：
- 各渠道数据割裂，无法形成完整客户画像
- 线索来源难以准确追溯，同一客户多渠道重复
- 无法判断哪个渠道真正有效，投放策略难以优化

#### 问题2：线索质量参差不齐

传统漏斗思维导致：
- 大量「无需求、无预算、无决策权」的无效线索混入
- 销售团队接到线索后要花大量时间筛选低质量线索
- 高价值线索被淹没在噪音中，错过最佳跟进时机

#### 问题3：信息不完整，需求模糊

客户留言往往是自然语言描述：
- 缺少结构化字段（预算、时间、决策链等）
- 需求表述模糊（「想了解一下」「先看看方案」）
- 销售需要反复沟通才能获取关键信息

### 2.2 营销售协同断层

#### 问题4：市场与销售互相甩锅

- **市场部委屈**：投入大量资源获取线索，销售跟进不力，浪费资源
- **销售部委屈**：线索质量差、意向不明，跟进效率低

根本原因：**缺乏统一的线索评估标准**

#### 问题5：交接信息丢失

销售拿到线索时：
- 不清楚线索的来源和培育过程
- 需要从头了解客户背景，损耗精力
- 客户在培育过程中的互动数据无法传递

#### 问题6：跟进节拍不统一

- 销售凭经验判断跟进时机
- 没有明确的 SLA（服务等级协议）
- 跟进频率参差不齐，容易错过关键节点

### 2.3 销售执行问题

#### 问题7：话术不统一，能力差异大

- 资深销售靠经验，新人靠运气
- 关键信息（产品能力、价格、交付周期）容易随口承诺
- 不同销售给出的方案和报价差异大，客户信任度下降

#### 问题8：合规风险高

销售过程中：
- 违规承诺（价格、周期、能力）可能导致罚款或客诉
- 质检员抽听录音比例不足 5%，大量违规话术事后才被发现
- 「100% 解决」「最低价」「一定能在 X 天内上线」等表述风险极高

#### 问题9：线索流失率高

- 销售只通过一次电话就决定是否丢弃线索
- 暂无意向的客户缺乏持续培育机制
- 大量潜在线索被放弃或转向竞争对手

## 三、行业最佳实践：线索评分体系

### 3.1 什么是线索评分（Lead Scoring）

线索评分是通过若干指标，从客户行为和属性评估线索质量，预测成交可能性的方法。

**核心价值**：
- 帮助销售集中精力在高价值线索上
- 让市场与销售对齐「合格线索」的定义
- 优化资源配置，提高投入产出比

### 3.2 评分维度

通常分为两大类：

| 维度类型 | 说明 | 示例 |
|---------|------|------|
| **显性指标** | 线索本身的数据属性 | 公司规模、行业、职位、预算 |
| **隐性指标** | 客户互动行为 | 浏览页面、下载资料、参加活动、回复邮件 |

### 3.3 评分模型示例

#### BANT 模型
- **B**udget（预算）：是否有明确预算范围
- **A**uthority（决策权）：是否是决策人或能影响决策
- **N**eed（需求）：需求是否具体、紧迫
- **T**imeline（时间线）：是否有明确的购买时间

#### 行为+属性双维度

| | 行为参与度高 | 行为参与度低 |
|--|------------|------------|
| **属性匹配高** | A1（最优先） | A4（需要培育） |
| **属性匹配低** | B1（属性待验证） | D4（暂不跟进） |

## 四、AI Agent 在营销线索场景的应用

### 4.1 AI 能做什么

根据行业调研，AI 在营销场景的应用方向：

| 应用方向 | 具体能力 | 价值 |
|---------|---------|------|
| **智能评分** | 自动分析线索，输出评分和优先级 | 减少人工筛选，提升效率 |
| **客户画像** | 自动提取关键信息，构建客户画像 | 减少信息缺失 |
| **跟进提醒** | 自动触发跟进任务，追踪最佳时机 | 避免遗漏 |
| **内容生成** | 生成个性化跟进话术和邮件 | 降低销售能力差异 |
| **数据分析** | 分析转化漏斗，识别优化点 | 数据驱动决策 |

### 4.2 已有实践案例

#### 案例1：国际制造企业 + MarketUP MA + AI Agent
- 线索从「低效流转」到「高效增长」
- 实现了线索效率大幅提升、营销售真正闭环

#### 案例2：得助 AI 销售助手
- 在销售开口之时化解合规风险
- 将质检从「事后追溯」变为「事中拦截」

#### 案例3：纷享销客 AI + CRM
- 结合营销服领域积累的海量数据和场景
- 提供行业化 AI 模型，支持个性化调优

## 五、Agent 与人工的责任边界设计

### 5.1 Agent 适合做的事

| 场景 | Agent 能力 | 边界 |
|------|-----------|------|
| **信息提取** | 从非结构化文本提取关键信息 | 只做整理，不做判断 |
| **评分计算** | 基于规则和数据计算线索分数 | 评分结果需人工确认 |
| **方案匹配** | 基于产品目录匹配推荐方案 | 不承诺，只建议 |
| **话术生成** | 基于 SOP 生成标准话术 | 不生成超出 SOP 的内容 |
| **风险提示** | 识别可能违规的表达 | 只提醒，不拦截 |
| **数据整理** | 汇总线索上下文，输出结构化报告 | 保持数据来源可追溯 |

### 5.2 必须人工做的事

| 场景 | 原因 | 边界 |
|------|------|------|
| **承诺价格/周期** | 涉及商业承诺和法律风险 | 必须基于正式报价单 |
| **决策是否跟进** | 涉及资源分配和商业判断 | Agent 只能提供建议 |
| **谈判和议价** | 涉及商业敏感信息 | 必须人工主导 |
| **处理异议和投诉** | 涉及客户关系维护 | 必须人工介入 |
| **高风险客户** | 涉及合规和信誉风险 | 需要人工复核 |

### 5.3 核心原则：人负责，AI 辅助

```
┌─────────────────────────────────────────────────────────┐
│                      商业责任链                          │
├─────────────────────────────────────────────────────────┤
│  线索获取 → Agent 整理评分 → 人工复核 → 人工决策跟进   │
│                         ↑                               │
│                   Agent 只能辅助                         │
│                   人工必须兜底                           │
└─────────────────────────────────────────────────────────┘
```

**关键设计原则**：

1. **Agent 不创造信息，只搬运和整理信息**
   - 所有产品参数必须来自产品目录（query_product）
   - 价格、周期等敏感信息必须标注「以正式报价单为准」

2. **Agent 输出必须可审计**
   - 展示推理过程（Trace）
   - 标注数据来源
   - 明确人工复核点

3. **Agent 必须识别自己的边界**
   - 当信息不足时，明确提示缺失
   - 当涉及承诺时，明确标注风险
   - 当判断不确定时，建议人工介入

## 六、对本项目的设计建议

### 6.1 产品能力边界

| 功能 | Agent 可做 | 必须人工 |
|------|----------|---------|
| 线索评分 | 基于 SOP 评分维度计算分数 | 最终评级确认 |
| 痛点分析 | 基于客户留言识别痛点 | 验证是否准确 |
| 方案推荐 | 基于产品目录匹配 | 不承诺效果 |
| 话术生成 | 基于 SOP 生成 | 不超出 SOP |
| 风险提示 | 识别缺失信息和违规表达 | 不拦截沟通 |

### 6.2 Trace 设计要求

每个 Agent 执行必须展示：
1. **推理摘要**：为什么这样做
2. **数据来源**：用了哪些产品资料/SOP
3. **结论边界**：哪些是 Agent 判断，哪些需要人工确认
4. **风险提示**：哪些信息缺失，哪些表达可能有风险

### 6.3 验证指标建议

如果用于真实业务，建议跟踪：

| 指标 | 说明 |
|------|------|
| 线索转化率 | Agent 评分高的线索是否真的转化更好 |
| 跟进响应时间 | Agent 辅助后是否缩短了响应时间 |
| 信息完整度 | Agent 是否减少了销售补全信息的次数 |
| 违规率 | Agent 是否减少了违规承诺的发生 |
