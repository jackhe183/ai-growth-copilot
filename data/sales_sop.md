# AI Growth Copilot 销售 SOP

## 一、线索评分标准

### 评分维度（总分 100 分）

| 维度 | 分值 | 评估标准 |
|------|------|----------|
| 意向强度 | 30分 | 高（20-30）：明确表达采购意向；中（10-19）：有需求但待明确；低（0-9）：初步了解 |
| 预算匹配 | 20分 | 明确（15-20）：提及预算范围；模糊（5-14）：未明确但有支付能力迹象；无（0-4）：未提及 |
| 时间紧迫性 | 20分 | 高（15-20）：明确时间节点；中（8-14）：有计划但不紧迫；低（0-7）：无时间要求 |
| 决策权 | 15分 | 明确（10-15）：本人可决策；影响（5-9）：可影响决策；未知（0-4）：未知 |
| 需求清晰度 | 15分 | 清晰（10-15）：问题具体、有明确痛点；模糊（5-9）：有方向但不具体；不清晰（0-4）：泛泛了解 |

### 等级划分

- **A级（80-100分）**：优先跟进，1个工作日内联系
- **B级（60-79分）**：重点跟进，3个工作日内联系
- **C级（40-59分）**：普通跟进，1周内联系
- **D级（0-39分）**：暂缓或培育，持续触达

## 二、信息收集清单

每次跟进必须确认以下信息：

### 必填信息
1. 公司名称和规模（人数）
2. 当前使用的系统/工具
3. 具体痛点和场景
4. 预算范围
5. 期望上线时间
6. 决策链条（谁决策、谁评估、谁使用）

### 选填信息
1. 当前客服/知识管理现状
2. 过往相关供应商询价情况
3. 对 AI 的了解程度

## 三、跟进话术模板

### 首次联系
```
您好，我是 [姓名]，来自 [公司] 的解决方案顾问。
看到您对我们的 [产品名] 感兴趣，想跟您做个简短沟通。
请问现在方便吗？

[如方便]
了解到您想 [客户原话中的需求]，我这边可以给您介绍一下我们的方案。
想先请教几个问题：
1. 您目前在这方面是怎么处理的？
2. 团队规模大概多少人？
3. 期望什么时候能看到效果？
```

### 方案介绍后
```
根据您的情况，我建议 [方案建议]。
优势在于：
1. [优势1，可量化]
2. [优势2，与客户痛点相关]
3. [优势3，与客户时间要求匹配]

关于费用，我们的标准方案是 [价格区间]，具体会根据 [定制因素] 来调整。
您看这个方向是否符合预期？
```

## 四、常见拒绝处理

| 拒绝类型 | 应对策略 |
|----------|----------|
| "没时间" | 询问合适的时间段，提供 15 分钟简短介绍 |
| "在比价中" | 了解其他供应商方案，强调差异化优势 |
| "预算不够" | 探讨分期方案或精简版方案 |
| "领导没批" | 了解决策流程，提供决策参考材料 |
| "不需要" | 了解现状，尝试发现潜在痛点 |

## 五、风险点识别

### 高风险特征
- 时间紧迫但需求模糊
- 多次变更需求
- 只问价格不关注方案
- 无法联系到决策人
- 预算明显低于市场价

### 风险应对
- 需求不清时，先做免费诊断
- 价格不匹配时，探索精简版或分期
- 决策链不清时，要求明确对接人
