# lead-scoring-followup

## 1. 概述

**Skill 名称**: lead-scoring-followup
**版本**: 1.0.0
**功能**: 对潜在客户线索进行评分、分析痛点、识别风险并生成跟进建议
**适用场景**: B2B 营销线索筛选和销售跟进决策支持

## 2. 输入

### 必需字段
- `customer_message`: 客户原始表达/留言
- `interested_product`: 客户感兴趣的产品/服务

### 可选字段
- `source`: 线索来源（官网表单/活动报名/社媒私信/邮件咨询等）
- `company`: 公司名称
- `industry`: 所属行业
- `sales_notes`: 销售备注信息
- `contact`: 联系信息（姓名、职位、电话、邮箱）

## 3. 工具

### query_product

查询本地 mock 产品/服务资料，用于获取产品能力、价格范围、交付周期等真实信息。

**输入格式**:
```json
{"type": "product", "query": "关键词"}
{"type": "product", "id": "产品ID"}
{"type": "sop", "query": "评分标准"}
{"type": "forbidden", "query": "价格承诺"}
{"type": "lead", "query": "关键词"}
```

**输出**: 返回匹配的产品信息/SOP 内容/禁止承诺事项

## 4. 执行流程 (ReAct)

### Step 1: Reasoning Summary
分析线索上下文，给出简短推理摘要和路由决策：
- 是否需要查询产品资料？
- 是否需要查询 SOP？
- 是否需要查询禁止承诺事项？
- 综合判断线索等级

### Step 2: Act
按需调用 `query_product` 工具：
- 查询匹配产品的详细信息
- 查询评分标准和推荐流程
- 查询禁止承诺事项（用于风险提示）

### Step 3: Observe
获取工具返回结果，分析：
- 产品能力与客户需求是否匹配
- 客户需求是否符合产品前提条件
- 潜在风险点

### Step 4: Answer
生成清晰分段的分析结果（见下方输出格式）

## 6. 约束规则

### 必须遵守
- **禁止编造**: 不得自行编造价格、交付周期、能力参数，必须基于 `query_product` 返回内容
- **引用来源**: 回答中涉及产品参数时，必须注明来源
- **合规提示**: 涉及价格、周期、能力时，必须提醒"以正式报价单为准"

### 评分原则
- 评分应客观，基于 SOP 中的评分维度
- 话术应符合销售 SOP 规范
- 风险识别应覆盖高风险特征
